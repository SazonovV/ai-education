Раздел 01

Что такое вайбкодинг

От идеи на естественном языке к коду без ручной реализации

Определение

Суть подхода

  • Пишете, что хотите получить
  • AI генерирует рабочее приложение
  • Вы проверяете, что работает, и даёте итеративные правки

Практический эффект

  • Порог входа близок к нулю
  • Вместо кода фокус на результате
  • Скорость принятия решений резко выше

"Забудь, что код существует. Опиши, что хочешь. Если сломалось — скажи AI, что сломалось."

Термин ввёл Андрей Карпатый — один из ведущих исследователей AI — в начале 2025 года.

Почему это взлетело

Низкий порог

Не нужны глубокие знания языков и фреймворков, чтобы получить прототип.

Скорость

От идеи до демо — минуты, а не спринты.

Доступность

Запуск прямо в браузере без локальной настройки окружения.

Видимый результат

Обратная связь сразу в интерфейсе, а не в документах.

Кто уже использует

Стартапы

Быстро проверяют идею до масштабных инвестиций.

Продакты и менеджеры

Делают интерактивные демо без отдельного цикла разработки.

Дизайнеры

Переходят от статичных макетов к живым прототипам.

Аналитики

Собирают дашборды и визуализации данных за минуты.

Разработчики

Ускоряют внутренние утилиты и технические эксперименты.

Хакатоны

Максимальный результат при минимальном времени.

Раздел 03

Живой пример

Один запрос -> короткие итерации -> рабочее приложение

Кейс: дашборд для анализа CSV

Запрос

"Сделай дашборд для анализа CSV: загрузка файла, таблица, фильтрация, 3 графика, тёмная тема."

Результат за ~30 секунд

  • Загрузка файла и таблица
  • Фильтрация и сортировка
  • Столбчатый, линейный и круговой графики
  • Рабочий интерфейс для демо

CSV — обычный формат таблиц, который умеет сохранять Excel. Дашборд можно опубликовать по URL за один клик.

Итеративный цикл вайбкодинга

Запрос 1

Базовый интерфейс

→

Запрос 2

Фильтры по колонкам

→

Запрос 3+

Тема, оптимизация, шаринг

Каждая итерация 10-30 секунд. Весь MVP часто укладывается в 5-15 минут.

Раздел 05

Вайбкодинг и другие режимы AI

Скорость vs контроль: важно выбирать подход под задачу

Три подхода

Вайбкодинг

"Сделай мне". Максимум скорости, минимум гарантий.

Агентский режим

"Сделай по правилам". Контролируемые действия и ревью.

Мультиагенты

Планирование, исполнение и проверка отдельными агентами.

Эти подходы не конкурируют, а дополняют друг друга на разных этапах: прототип → разработка → масштабирование.

В чём разница: вайбкодинг vs агентский режим

КритерийВайбкодингАгентский режим
КонтрольМинимальныйВысокий (через правила)
Проверка"Потыкали руками"Тесты + дифф + policy
ПовторяемостьНизкаяВысокая
Подходит для продакшенаОграниченноДа, при настроенных guardrails

Как повысить качество вайбкодинга

Spec first

Сначала попросите AI написать спецификацию. Проверьте. Только потом — команда делать.

Git checkpoints

Вставьте промпт-правило для Git в начале работы — AI будет коммитить после каждого шага.

Без секретов

Вставьте промпт-правило для секретов — AI будет использовать .env и не будет просить пароли.

Self-check

Попросите AI проверить edge-cases: что если данных нет? Данных слишком много? Ввод некорректный?

Промпт-правило: Git

Скопируйте и вставьте в начале работы с AI-инструментом

Правила работы с Git в этом проекте:

1. После каждого успешного изменения — делай git add
   и git commit с понятным сообщением на русском.
2. Не объединяй несколько изменений в один коммит.
   Одна задача = один коммит.
3. Перед рискованным изменением — сначала сделай коммит
   текущего рабочего состояния.
4. Если что-то сломалось — не чини поверх. Скажи мне,
   и я сделаю git checkout -- . для отката.
5. Не коммить .env, node_modules, dist.

Промпт-правило: секреты

Скопируйте и вставьте в начале работы с AI-инструментом

Правила работы с секретами в этом проекте:

1. Никогда не вставляй пароли, токены или API-ключи
   прямо в код. Используй переменные окружения.
2. Все секреты — в файле .env. Я создам его сам.
3. В коде: process.env.ИМЯ_ПЕРЕМЕННОЙ.
4. Файл .env должен быть в .gitignore. Проверь.
5. Если нужен ключ — не придумывай. Напиши мне,
   какую переменную ты ожидаешь.
6. Создай .env.example с названиями переменных
   без значений.

Путь: от прототипа к продукту

1. Вайбкодинг

Быстрый прототип и проверка идеи за минуты

2. Оценка

Решаем: идём в продукт или идея не взлетела?

3. Агентский режим

«Боевая» реализация по стандартам, используя прототип как образец

4. Стандартизация

Закрепляем rules / skills / tools для всей команды

Прототип — это наглядное ТЗ, записанное языком работающего приложения, а не финальный production-код.

Раздел 06

Практические рекомендации

Когда использовать, как общаться с AI и что попробовать прямо сейчас

Когда стоит вайбкодить (чеклист)

  • ✓ Прототип или демо для стейкхолдеров
  • ✓ Внутренний инструмент команды
  • ✓ Одноразовая автоматизация
  • ✓ Быстрый эксперимент с новой технологией
  • ✓ Лендос / маркетинговая страница
  • ✓ Хакатон и быстрый proof-of-concept

Полезные шаблоны запросов

# Начало проекта
"Создай [что нужно]. Функции: [1], [2], [3]. Дизайн: [стиль]."

# Доработка
"Добавь [функцию]. Не трогай [критичные части]."

# Исправление
"Когда я [действие], происходит [проблема]. Должно быть [ожидание]."

# Brainstorm
"Предложи 3 варианта решения с разной сложностью и сроком."

Чем конкретнее контекст, ограничения и критерии готовности, тем меньше итераций.

Superprompt: чем больше деталей в одном сообщении (для кого, что должно уметь, как выглядеть, что НЕ нужно), тем точнее результат.

Попробуй сам: готовые задания

Откройте bolt.new, вставьте промпт, посмотрите результат. Каждое задание — 5-10 минут.

1. Таймер для стендапов

Большой таймер на 15 мин, список участников, кнопка «следующий», мигание при истечении времени. Тёмная тема, крупный шрифт.

Затем: «Добавь звук, когда остаётся 1 минута»

2. Калькулятор стоимости проекта

Форма: тип проекта, кол-во экранов, авторизация, БД, дизайн. Результат: стоимость, сроки, разбивка по этапам.

Затем: «Добавь сравнение двух вариантов рядом»

3. Трекер привычек

Список привычек с галочками на 7 дней, счётчик серии, прогресс-бар. Данные в браузере, без регистрации.

Затем: «Добавь переключение между неделями»

4. Генератор еженедельного отчёта

Форма: сделано, планы, блокеры, настроение. Результат: красивый отчёт, кнопка «скопировать для Slack», история.

Затем: «Добавь режим руководителя: все отчёты на одной странице»

Как работать с заданиями

Шаг 1

Скопируйте промпт и вставьте в bolt.new

→

Шаг 2

Посмотрите результат и попробуйте доработки

→

Шаг 3

Сохраните или сломайте — проверьте границы

Цель — не идеальный результат с первого раза, а навык итеративного диалога с AI. Чем точнее промпт, тем меньше доработок.
